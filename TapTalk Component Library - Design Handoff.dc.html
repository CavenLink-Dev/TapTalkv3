Design system summary

TapTalk’s visual language is one calm sky-blue over cool off-white surfaces. No gradients, no textures, no playfulness. Generosity comes from radius and whitespace, not from color or decoration. The Fitzgerald-key palette is reserved for AAC symbol cells — never used as decoration anywhere else.

TapTalkLogo
In-app brand mark · SVG · pure geometry.
TapTalk
Mark · light
Mark · dark
On brand
Construction · 64 grid
Two elements: a tap-dot (r=10.5) and an arc above it (quadratic, stroke 9.5, round caps). The arc’s right anchor lands roughly above the dot’s right edge so the silhouette reads as voice projecting from a tap. No third stroke. No wordmark embedded in the mark — wordmark is a separate composition.
PurposeSingular brand mark for in-app surfaces (splash, header, empty states, settings). Communicates the core idea of TapTalk — a single tap producing voice — in a calm, adult, geometric form.
Visual designTwo-element SVG on a 64×64 grid: solid tap-dot (r=10.5, x=22, y=46, fill colors.primary) and a communication arc stroked above (Q(32,6) from x=11 to x=55, stroke-width 9.5, round caps, no fill). Single color. Mark is the same in dark mode but tinted primaryPressed for legibility on near-black. Wordmark is set in SF Compact Rounded Heavy (800), letter-spacing −1.5px, with the mark scaled 0.72× of cap-height, sitting left of the “T”.
StatesStatic (default), Splash reveal (arc draws, dot pops — see HeroAnimation), Loading proxy (arc only, gently breathing). Never tilted, never colored outside palette.
AccessibilityWrapper View has accessibilityRole="image" and label “TapTalk”. SVG children are accessibilityElementsHidden. Contrast ratio of the mark against background 4.55:1 on light, 6.2:1 on dark.
MotionSplash only: arc strokeDashoffset draws 0 → length over 520ms easeStandard; dot scales 0 → 1 with easeSpring 300ms, starting at +180ms. Idle on splash: arc translateY ±2px, 1400ms loop (durFloat).
Reduced motionMark renders fully drawn at opacity 0; cross-fades to 1 over 200ms. No drawing, no float.
Props / API{ size?: number; // default 56 tone?: 'light' | 'dark' | 'onBrand'; // default 'light' wordmark?: boolean; // default false animated?: boolean; // splash reveal testID?: string }
Implementationreact-native-svg primitives (Svg / Path / Circle). For animated reveal use Reanimated useSharedValue for strokeDashoffset on the arc (compute the path length once via ~78), drive dot scale via the same hook. Do not animate the wordmark.
Exports needed laterApp icon (1024×1024 PNG, generated from the same SVG with a 18% padding on a #FFFFFF background, primary-fill mark). iOS adaptive variants at 60·76·83.5·1024pt. Splash (2732×2732 PNG, mark centered 28% of width on #F1F5F9). Generate via a `scripts/build-icons.ts` step using `sharp` (already a transitive dep) or manual export; do not commit PNGs hand-edited.
RisksArc thickness reads as a smile at small sizes (<24pt) — clamp size ≥ 28 in UI. App Store icon must NOT include the wordmark (HIG). Dark-mode tint must not drop below 4.5:1 contrast.

Purpose
The single dominant action per screen. Always exactly one primary visible at a time. Bottom-docked in flows, inline only in dense forms.
Visual
Height 54pt (min 50, hit 56). Radius 10. Label SF Compact Rounded Heavy 17/700, letter-spacing −0.3. Padding 18 horizontal. Primary fill primary; secondary softBlue+primaryDark label; ghost transparent+border 1.5px; danger danger. No ALL-CAPS. Adult typography prefers sentence-case button labels.
States
default · pressed (scale 0.97 + fill→primaryPressed + shadow pop) · loading (LoadingDots inside, button locked) · disabled (fill progressTrack, label textTertiary) · success (one-shot, see SuccessAnimation).
Accessibility
accessibilityRole="button", required accessibilityLabel prop. Disabled = accessibilityState={`{disabled:true}`}. Loading announces “Working…” via accessibilityLiveRegion="polite". Contrast: primary 4.66:1, danger 4.51:1.
Motion
Press in: scale 1 → 0.97, fill cross-fade primary→primaryPressed, 120ms easeStandard. Release: scale 0.97 → 1, fill back, 180ms easeSpring. Haptic selection on press in. Loading dots: 3 bars stagger 0/220/440ms (existing LoadingDots tinted onDark).
Reduced motion
No scale, no shadow change. Only opacity 1 → 0.85 → 1 on press (180ms total). Loading dots fall back to a single static pulsing dot at 0.6 Hz.
Props / API
{ label: string; onPress: () => void; accessibilityLabel: string; variant?: 'primary'|'secondary'|'ghost'|'danger'; size?: 'lg'|'md'; // md = 44pt (inline) loading?: boolean; disabled?: boolean; leadingIcon?: IoniconsName; fullWidth?: boolean; // default true testID?: string }
Implementation
Replace existing src/components/native/PrimaryButton.tsx. Use Pressable + Reanimated useAnimatedStyle for scale; expo-haptics for feedback. Hit slop 8 on all sides if size=md. Read Reduce Motion once via AccessibilityInfo and gate the worklet.
Risks
Two primary buttons on one screen breaks the system — enforce in code review. Loading state must lock taps (an extra tap during async work is a real bug we have today). Dynamic Type at XXL pushes ghost label past container — clamp label to one line with numberOfLines={1} + adjustsFontSizeToFit (min 13).

Purpose
Single- or multi-select card for high-stakes choices: app mode, role, theme, text size, age range. A bigger, opinionated alternative to plain radio rows.
Visual
White surface, 16 radius, 1.5px border. Selected: border swaps to 2px primary + pop shadow. Leading icon 48×48 in 12-radius tint chip. Title 17/800. Description 13/400 textMuted. Trailing 24×24 selection indicator (checkmark when selected, hairline circle when not). Min height 76pt.
States
default · pressed (scale 0.98, background→background) · selected · selected-pressed (slight scale, retain pop) · disabled (opacity 0.55, no haptic, no press response) · error (border→danger, helper text under group).
Accessibility
accessibilityRole="radio" (or “checkbox” when multi), accessibilityState={`{selected, disabled}`}. Description joined with title in accessibilityLabel. Group parent uses accessibilityRole="radiogroup" with the question as the label. Selection conveyed by checkmark icon, not color alone.
Motion
Press: scale 0.98, 120ms easeStandard. Selection: border-width 1.5→2 + color cross-fade 200ms; check icon scales 0→1 with easeSpring 260ms; shadow fades in over 260ms. Deselect mirror. Haptic light on select.
Reduced motion
No scale, no spring. Border and shadow snap. Check icon fades 0→1 over 120ms.
Props / API
{ title: string; description?: string; icon?: IoniconsName | (() => ReactElement); selected: boolean; onPress: () => void; disabled?: boolean; multi?: boolean; // checkbox semantics accessibilityLabel?: string; // computed if omitted testID?: string }
Implementation
New file src/components/native/SelectableCard.tsx. Replaces ad-hoc cards in app mode, theme, and consent flows. Pair with <SelectableCardGroup> wrapper that owns selection state and applies the radiogroup role.
Risks
Long descriptions cause cards in a group to grow unequally — set numberOfLines={2} on description; truncate with “…”. At Dynamic Type XXXL the icon chip steals width; switch to vertical layout when accessibility text scale ≥ 1.3.

Purpose
The default container for grouped content. Every screen-level surface that isn’t a list or sheet is a Card.
Visual
Three variants share one component, controlled by radius: standard (16, padding 16), hero (40, padding 24-28), compact (16, padding 14/16, row layout). All carry a 1.5px border hairline AND card shadow — both, not either. White fill.
States
static · pressable (scale 0.985 on press, no fill change) · raised (shadow → cardRaise, for bottom-attached actions) · promotional (shadow → pop, sparingly — paywall, hero CTAs).
Accessibility
Decorative cards are containers with no role. pressable mode adds role="button" + required label. Cards group their children for VoiceOver via accessible={true} when contents are read as a unit (e.g. summary card).
Motion
Pressable: scale 1 → 0.985, 180ms easeSpring. Entrance (when used in a fresh screen): translateY 12 → 0 + opacity 0 → 1, 260ms easeStandard, staggered 60ms per card.
Reduced motion
No press scale. Entrance fades only (no translate), staggered 80ms.
Props / API
{ children: ReactNode; radius?: 'standard'|'hero'|'compact'; // default standard elevation?: 'card'|'raise'|'pop'|'none'; padding?: number; // override default onPress?: () => void; // makes it pressable accessibilityLabel?: string; // required if onPress style?: ViewStyle; testID?: string }
Implementation
Replace src/components/native/Card.tsx. Map radius to radii tokens; map elevation to shadows. For Android, apply elevation integer separately. Do not nest pressable cards.
Risks
Cards inside cards quickly destroy the visual hierarchy — lint warning if depth > 1. The pop shadow is reserved; reserve for ≤ 1 element per screen.

Purpose
Two modes: segmented for discrete multi-step flows (onboarding, registration), single for continuous progress (password strength, file upload, loading).
Visual
Track 7pt tall, pill 22 radius, fill progressTrack, fill color primary (or tone). Segmented: equal-flex segments with 5pt gap. Optional eyebrow label “Step N of M” and trailing % or numeric value. Complete state turns ALL segments success as a moment of celebration.
States
empty · advancing · complete · indeterminate (single-bar only — see LoadingDots if length unknown but discrete). Strength tones: danger / warning / success.
Accessibility
role="progressbar". accessibilityValue={`{min:0, max:totalSteps, now:currentStep, text:'Step 5 of 8'}`}. Step changes use accessibilityLiveRegion="polite". Strength bar must include a textual label — color alone is never the signal.
Motion
Segmented advance: new segment fills left-to-right via width 0% → 100% over 360ms easeStandard, with a 45ms catch-up stagger if multiple segments are filled at once (already implemented in current code). Complete: all-segments color cross-fade primary → success over 220ms then a single pulse scale 1 → 1.03 → 1 over 260ms.
Reduced motion
Segments fill instantly with opacity cross-fade only (220ms). No pulse on completion — color cross-fade only.
Props / API
{ mode: 'segmented' | 'single'; // segmented: currentStep?: number; totalSteps?: number; // single: value?: number; // 0..1 tone?: 'primary'|'success'|'warning'|'danger'; // shared: showLabel?: boolean; label?: string; // override “Step N of M” testID?: string }
Implementation
Extend existing src/components/native/SegmentedProgressBar.tsx with mode branch + single-bar implementation. Keep existing Reanimated worklet for fill width.
Risks
If totalSteps > 12 the segments become indistinguishable. Cap at 12; switch to single-bar mode beyond that.

Purpose
Replaces the iOS native spinner everywhere — the existing app already uses pulsing dots; we formalize it as a single component with three contexts.
Visual
Three dots, separation = dot diameter. Standalone: 12pt dots, primary fill, softBlue capsule (padding 10v/18h, radius 24). onPrimary: 8pt white dots on primary buttons, no capsule. Inline: 7pt dots in current text color, no capsule, for text rows.
States
animating only (component is unmounted otherwise). No hover or pressed.
Accessibility
accessibilityRole="progressbar" with accessibilityState={`{busy:true}`} and label “Loading” (or caller-provided). When inside a button the parent button takes the label and the dots are hidden via accessibilityElementsHidden.
Motion
Each dot opacity 0.3 → 1 → 0.3 over 800ms, staggered 0/220/440ms. Mild scale 0.8 → 1.15 in parallel.
Reduced motion
A single dot pulses opacity 0.5 → 1 → 0.5 at 0.6 Hz. No scale, no stagger. Or use a static “Loading…” text with bouncing ellipsis (text role).
Props / API
{ variant?: 'standalone' | 'onPrimary' | 'inline'; // default standalone size?: number; // override dot size color?: string; // override fill accessibilityLabel?: string; // default 'Loading' testID?: string }
Implementation
Extend existing src/components/LoadingDots.tsx (already implements the staggered pulse) — add variant branch and Reduce Motion fallback (currently missing). Move to src/components/native/ for consistency.
Risks
When loading exceeds 3s, silently swap the label “Loading” → “Still working…” via the parent at 3s, then “This is taking longer than usual” at 8s, to set expectations.
I
Skeleton
Placeholder geometry for hydrating screens. Calm shimmer, not flash.
Purpose
Holds layout while content hydrates from AsyncStorage / SQLite. Mirrors the shape of incoming content — never generic.
Visual
Three primitives: SkeletonRect, SkeletonLine, SkeletonCircle. Base color input #EAEEF2, shimmer highlight progressTrack #D5E1E8. Line radius 6, default line widths 100% / 75% / 50% / 35% rotating per row to feel natural. NEVER use a fully grey card with a single block — always granular.
States
shimmering (default) · paused (Reduce Motion) · fadeOut (250ms when real content arrives — cross-fade, do not snap).
Accessibility
Top-level Skeleton wrapper announces “Loading content” once via accessibilityLiveRegion="polite". Children are accessibilityElementsHidden + importantForAccessibility="no-hide-descendants" so VoiceOver doesn’t enumerate empty boxes.
Motion
Shimmer gradient slides translateX −100% → +100% over 1600ms linear, infinite. Gradient is a 200%-wide three-stop: base / highlight / base. Use a single Reanimated shared value at the Skeleton group root and pass it to children, not one loop per box.
Reduced motion
No shimmer. Static fill at #E1E7ED (between base and highlight). Wrapper announces loading state and the skeleton replaces with content on arrival (no fade either — would be motion).
Props / API
<Skeleton> // group, owns the shared shimmer <Skeleton.Rect width height radius /> <Skeleton.Line width? height=12 /> <Skeleton.Circle size /> </Skeleton> // + presets: <Skeleton.Preset kind='card-row' count={3} /> <Skeleton.Preset kind='symbol-grid' rows={4} cols={5} />
Implementation
New file src/components/native/Skeleton.tsx. Use react-native-svg LinearGradient inside <Rect> for the shimmer (no extra dep). Context-passed shared value so all children animate in lock-step.
Risks
Skeletons that don’t match real content shape feel worse than a spinner — every screen with a skeleton MUST ship the matching preset. Cap skeleton lifetime to 5s and fall back to LoadingDots if hydration takes longer (something is wrong).
